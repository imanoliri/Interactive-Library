# Interactive Library Contents - Technical Reference

This document provides a technical overview of the "Contents" mini-games within the Interactive Library project. These 8 applets are standalone, data-driven web applications designed to run within the context of an interactive book, utilizing JSON data extracted during the book's compilation phase.

## 1. Architecture Overview

### Data Injection Pipeline
The mini-games are entirely decoupled from the core Python compilation engine (`generate_books.py` and `interactive_book_from_html.py`). Instead, they rely on pre-compiled data "artifacts" generated by the engine.

When `generate_books.py` processes a source HTML file, it extracts specific metadata and serializes it into standard JSON formats. The mini-games then asynchronously `fetch()` these JSON files upon initialization.

**Key Serialized JSON Data Sources:**
*   `interactive_book_images.json`: An array of all image file names in the book (used by Memory and Order Images).
*   `interactive_book_word_count.json`: An object mapping canonical words to their frequency count (used by Word Cloud and Anagrams).
*   `interactive_book_parapragh_texts.json`: An array of strings representing narrative paragraphs (used by Fill the Gaps and Word Cloud snippets).

### Design System
All mini-games follow a unified "Classic Simple" design system. This ensures visual consistency with the library's original applets while maintaining a clean, professional appearance.

**Core Tokens (`shared_game_styles.css`):**
*   **Theme**: Light theme (#f4f4f4 background).
*   **Layout**: Centered card (`.container`) with simple shadows.
*   **Typography**: System sans-serif (Arial).
*   **Colors**: Minimalist palette (White/Grey/Blue).

## 2. Applet Implementation Details

### 2.1 Anagrams (`/anagrams`)
*   **Concept**: A fast-paced word unscrambling game using vocabulary from the book.
*   **Data Source**: `interactive_book_word_count.json` (Keys are extracted via `Object.keys()`).
*   **Core Logic**:
    *   **Shuffling (Fisher-Yates)**: The Javascript rigorously shuffles strings to present the anagram, ensuring the scrambled word is never identical to the original word.
    *   **Normalization**: User input and target words are scrubbed of punctuation and lowercased via `cleanWord()`.
    *   **State**: Tracks `score` (persistent via `localStorage`), `streak`, and `hintIndex`.
    *   **Hints**: The hint system pre-fills the input field with sequential letters of the target word, deducting potential points on success.

### 2.2 Word Cloud (`/word_cloud`)
*   **Concept**: A visual representation of the book's most frequent vocabulary, rendered dynamically.
*   **Data Source**: `interactive_book_word_count.json` and `interactive_book_parapragh_texts.json`.
*   **Core Logic**:
    *   **Dependencies**: Relies heavily on external libraries `d3.js` (v5) and `d3.layout.cloud.js`.
    *   **Scaling**: Word sizes use a discrete linear domain scale based on frequency, bounded geometrically by `Math.sqrt(width * height)`.
    *   **Context Snippets**: Clicking a rendered SVG text element triggers a Regex search (`\b${word}\b`) against the paragraph array to fetch contextual occurrences, which are then passed to a secondary HTML container (`#snippet-container`).

### 2.3 Fill the Gaps (`/fill_the_gaps`)
*   **Concept**: A reading comprehension test where random words are removed from a paragraph.
*   **Data Source**: `interactive_book_parapragh_texts.json`.
*   **Core Logic**:
    *   **Paragraph Selection**: The JS filters the raw array for substantial paragraphs (length > 15 words) before picking one randomly.
    *   **Gap Generation**: Iterates through the selected string, identifies eligible target words (length >= 4 chars, no HTML tags), and replaces them with `<input>` fields sized dynamically (`width: ${cleanWord.length + 1}ch`).
    *   **Validation**: Real-time event listeners on the inputs provide instantaneous feedback class-toggling (`.correct` or `.wrong`).

### 2.4 Sudoku (`/sudoku`)
*   **Concept**: A classic 9x9 grid puzzle completely independent of the book's narrative data.
*   **Core Logic**:
    *   **State Management**: Tracks 1D integer arrays (`grid`, `solvedGrid`, `initialGrid`) tracking numbers 0-9. A parallel `Array of Sets` structure handles the "Pencil" notes mode.
    *   **Generation Engine**: Implements a randomized backtracking algorithm `fillGrid(g)` to generate a fully solved board, then randomly punches out `N` indices based on the active difficulty slider.
    *   **Conflict Resolution**: Does not aggressively prevent illegal moves. Instead, it highlights conflicting cells via `hasConflict(idx)` if multiple values sit in the same row/col/3x3 sector, enforcing user-driven deduction over machine restriction.
    *   **Keyboard Support**: Fully supports arrow-key navigation and number-key input, binding grid deltas to the `selectedCell` integer.

### 2.5 Memory Images (`/memory_images`)
*   **Concept**: A standard pair-matching visual test using graphical assets from the text.
*   **Data Source**: `interactive_book_images.json`.
*   **Core Logic**:
    *   **CSS Architecture**: Heavily leverages CSS 3D Transforms (`perspective: 1000px`, `transform-style: preserve-3d`, `backface-visibility: hidden`) rather than JS swapping, rotating the `.card` container on the Y-axis.
    *   **State**: Tracks `flippedCards[]` (max length 2) and `matchedPairs`. Utilizes minimal `setTimeout` delays to handle fail-state animations (flipping cards back over).
    *   **Deck Building**: Slices the asset array based on slider difficulty, duplicates the subarray, and performs a Fisher-Yates shuffle.

### 2.6 Order Images (`/order_images`)
*   **Concept**: A chronological narrative test. Players must click randomized images in the order they were encountered in the book.
*   **Data Source**: `interactive_book_images.json` (Inherent chronological order).
*   **Core Logic**:
    *   **Sequence Selection**: Extracts a sequential slice of $N$ images from the JSON to form the "True Sequence". It then clones and shuffles these objects for presentation.
    *   **Geometric Rendering (Circular)**: Instead of a linear flexbox, images are absolutely positioned using trigonometric calculation. Based on `(i / shuffled.length) * (2 * Math.PI)`, it determines the $X, Y$ coordinates around a predetermined center radius to form a perfect circle.
    *   **Validation**: The JS compares the clicked node's `data-path` against the active index of the `correctSequence` array.

### 2.7 Writing Table (`/writing_table`)
*   **Concept**: A complex UI allowing users to form structural words by dragging graphical letters into predetermined slots.
*   **Core Logic**:
    *   **DOM Generation**: Due to the massive number of necessary DOM nodes, the HTML is kept sparse, relying on `writing_table.js` to iterate arrays (`vowels`, `consonants`) and computationally `appendChild()` the UI elements to a CSS Grid.
    *   **Drag-And-Drop API**: Relies entirely on the native HTML5 standard (`draggable="true"`, `ondragstart`, `ondragover`, `ondrop`). Data payloads (`e.dataTransfer.setData`) transfer the literal character string.
    *   **Validation**: Iterates over CSS row-selectors to assemble string lengths and checks them asynchronously against dictionary functions.
    *   **Speech Synthesis**: Implements `window.speechSynthesis.speak(new SpeechSynthesisUtterance(text))` as an auditory accessibility feature.

### 2.8 Battle (Standalone) (`/battle`)
*   **Concept**: A detached, analytical interface for the main book's "Paths of Magic" engine, allowing simulated manipulation of elemental and faction variables.
*   **Core Logic**:
    *   **Architecture**: Modular ES6 system (`<script type="module">`). It breaks functionality into discrete imports (`draw.js`, `units.js`, `networks.js`).
    *   **Data Loading**: Uses complex `Promise.all()` structures to deeply scan the directory structure (`auto_data/nodes.json`, `<faction>_interactions.json`) to programmatically load the state of specific battle campaigns.
    *   **D3 Visualization**: Uses extensive SVG manipulation to plot relational "networks" (representing the magic circle combat mechanics) layered over a geographic map coordinate system `(X,Y)` defined in the master metadata.

### 2.9 Crossword Puzzle (`/crossword`)
*   **Concept**: A dynamic crossword generator using vocabulary from the book.
*   **Data Source**: `interactive_book_word_count.json`.
*   **Core Logic**:
    *   **Grid Placement**: Implements a recursive intersection-first placement algorithm. It starts with the longest word and attempts to cross subsequent words at shared characters, maintaining a valid grid structure.
    *   **Bounding Box**: Dynamically calculates the minimum bounding box of placed words to crop the grid for rendering.
    *   **Input Handling**: Supports auto-advancing focus to the next cell upon typing and backspace-driven reversal.

### 2.10 Quote Scramble (`/quote_scramble`)
*   **Concept**: A drag-and-drop game to reconstruct scrambled sentences from the book.
*   **Data Source**: `interactive_book_parapragh_texts.json`.
*   **Core Logic**:
    *   **Sentence Extraction**: Parses paragraphs into sentences using punctuation markers, filtering for those within a specific word count range (6-14 words).
    *   **Drag-and-Drop**: Uses the HTML5 Drag and Drop API with a "ghost" element to allow reordering words in a target drop zone.
    *   **Validation**: Compares the final stringified order of the DOM nodes against the original extracted sentence array.

### 2.11 Chapter Quiz (`/chapter_quiz`)
*   **Concept**: A reading comprehension game where players identify which chapter a specific snippet belongs to.
*   **Data Source**: `story_by_chapters.json`.
*   **Core Logic**:
    *   **Snippet Selection**: Randomly selects a chapter and extracts a 1-2 sentence snippet from its HTML content (stripping tags).
    *   **Distractor Generation**: Fetches other chapter titles to serve as incorrect multiple-choice options.
    *   **Scoring**: Tracks session-based score and total attempts in local memory.

### 2.12 Hangman (`/hangman`)
*   **Concept**: A vocabulary-guessing game where failed attempts deplete "magical energy".
*   **Data Source**: `interactive_book_word_count.json`.
*   **Core Logic**:
    *   **State**: Tracks `guessedLetters[]` and `livesLeft`.
    *   **Keyboard Integration**: Listens for physical keyboard `keydown` events in addition to virtual on-screen buttons.
    *   **Visual Feedback**: Uses CSS classes (`revealed`, `correct`, `wrong`) to style the letter slots and virtual keys dynamically.

### 2.13 Word Search (`/word_search`)
*   **Concept**: A classic hidden word grid populated with book vocabulary.
*   **Data Source**: `interactive_book_word_count.json`.
*   **Core Logic**:
    *   **Grid Population**: Randomly places words in 8 possible directions (horizontal, vertical, diagonal). It uses a collision-detection check before committing a word to the grid.
    *   **Selection Logic**: Implements a "drag-to-select" interface. It calculates the vector between the mouse-down and current hover position to lock the selection to a straight line.
    *   **Reverse Matching**: Validates selections in both forward and reverse directions to support backwards-placed words.

### 2.14 Typing Race (`/typing_race`)
*   **Concept**: A speed-typing challenge using narrative blocks from the text.
*   **Data Source**: `interactive_book_parapragh_texts.json`.
*   **Core Logic**:
    *   **Real-time Analytics**: Calculates WPM (Words Per Minute) using the standard 5-character-per-word formula and live accuracy percentage.
    *   **Visual Diffing**: Renders the target text with character-by-character color-coding (green for correct, red for wrong) as the user types.
    *   **Accessibility**: Auto-focuses the input area and supports re-starting with new text blocks.

## 3. General Principles and Best Practices

*   **Responsive Fallbacks**: When calculations (like word cloud fonts) are screen-dependent, `window.addEventListener('resize')` is used carefully in tandem with debouncing or complete re-initialization routines to prevent UI distortion.
*   **Asset Paths**: Due to the build step moving scripts to explicit internal book folders (`books/<BookName>/index.html`), all asset fetching paths are structurally relative (`./../../`) referencing the compiled root artifacts.
*   **Native Modals**: Rather than relying on external libraries like Bootstrap/SweetAlert, modals use a custom, accessible overlay system driven by `z-index` and basic flex centering.

## 4. Design & Styling (Classic Simple)

The mini-games follow a **Classic Simple** aesthetic. This design language prioritizes functional clarity and minimalist layout over modern decorative effects.

### AI CSS Generation Prompt
Use the following prompt to generate new game styles that match this aesthetic:

> "Generate a CSS file for a mini-game that follows a 'Classic Simple' aesthetic. The style must be minimalist and functional, avoiding glassmorphism, heavy glows, or vibrant gradients.
> 
> Key Requirements:
> 1. **Body**: Light grey background (`#f4f4f4`), centered layout using flexbox.
> 2. **Container**: White background card, centered, with `3rem` padding, `0.5rem` radius, and a very faint shadow (`0 0 2rem rgba(0,0,0,0.05)`).
> 3. **Typography**: Arial or sans-serif, standard weights.
> 4. **Buttons**: Flat Blue (#007BFF) with darker blue hover (#0056b3), `0.25rem` radius.
> 5. **Interactive Elements**: Use `1px solid #ccc` for borders, `#f9f9f9` for secondary backgrounds (like code blocks or displays), and standard semantic colors for feedback (Green for success, Red for error).
> 6. **Navigation**: Simple text links (`.nav-link`) in grey (#666) with underline on hover."
